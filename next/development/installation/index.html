Development environment installation

Prerequisites

  • Docker
  • Node.JS (>= 10)
  • Python (>= 3)
  • Yarn (>= 1.16)

Installation of dependencies (Ubuntu 18.04)

$ sudo apt-get install nodejs python3 python3-pip
$ sudo curl -sS https://dl.yarnpkg.com/debian/pubkey.gpg | sudo apt-key add -
$ sudo echo "deb https://dl.yarnpkg.com/debian/ stable main" | sudo tee /etc/apt/sources.list.d/yarn.list
$ sudo apt-get update && sudo apt-get install yarn

Docker stack

As OpenCTI has a dependency to ElasticSearch, you have to set the vm.max_map_count before running the containers, as mentioned in the ElasticSearch documentation.

$ sysctl -w vm.max_map_count=262144

Clone the latest version of the dev docker compose and start

$ git clone https://github.com/OpenCTI-Platform/docker.git
$ cd docker
$ docker-compose -f ./docker-compose-dev.yml up -d

Clone the project

$ git clone https://github.com/OpenCTI-Platform/opencti.git
$ cd opencti

Application dependencies

Install the API dependencies

$ cd opencti-platform/sopencti-graphql
$ yarn install

Install the frontend dependencies

$ cd ../opencti-front
$ yarn install

Install the worker dependencies

$ cd ../../opencti-worker
$ pip3 install -r requirements.txt

Config and run

GraphQL API

Configure

$ cp config/default.json config/development.json

By default the configuration match the docker stack configuration. You just need to change the user part:

"admin": {
  "email": "admin@opencti.io",
  "password": "ChangeMe",
  "token": "ChangeMe"
}

Start

$ cd opencti-graphql
$ yarn start

The first execution will create and migrate the schema.

Worker

Configure

$ cd opencti-worker
$ cp config.yml.sample config.yml

Change the config.yml file according to your

Start

$ python3 worker.py &

Frontend

Start

$ cd opencti-frontend
$ yarn start

Build for production use

Build the application

$ cd opencti-frontend
$ yarn build
$ cd ../opencti-graphql
$ yarn build

Start the production package

$ yarn serv
